de CQ18765 - Proceso Especial Batch Ripley

Objetivo: Generar un proceso (macro y programas COBOL) sh-PE-Ripley que para un set de identificaciones ("base") genere un archivo de salida con campos adicionales relacionados con cada Id.

La base para generar el informe por identificaciones puede realizarse de la extracción de datos internos o de datos proporcionados por el cliente.

Si el cliente manda la base, o sea Base Externa de ID. Se procesa y no hay ningún suscriptor de referencia, simplemente se refleja lo que se pide.

Si es con base interna debe aplicarse el estándar de extracción por NIT o Suscriptor de identificaciones del ICMCRECOPY.DAT

El proceso a crear debe tomar como proceso guia el icestd81.cbl

Se lee un archivo de ids con una de las estructuras siguientes:

  1. Archivo 1-11: Tipo ID (1) y Número ID (11)
  2. Archivo 1-11-45: Tipo ID (1) , Número ID (11), Nombres y Apellidos o Razón Social (45)

Se procesan con el PESVNO, que genera un archivo de ids válidas y otro de incosistentes. El proceso PESVNO ya existe.

Valores por defecto

Se deben definir los valores por defecto por cada uno de los parámetros listados, cada parámetro puede ser cambiado y los parámetros usados se deben guardar en un archivo de parámetros de mantenimiento en $DATOS

Salidas

El proceso batch debe incluir como mensajes de salida:

  • Estadísticas de Validación (PESVNO)
  • Estadísticas de cálculo de Score (SCOTBATCH)
  • Nombres de los archivos generados
Archivo log

Por cada procedimiento se debe registrar la siguiente información:

  • Parámetros Procesamiento
    • Archivo de parámetros
  • Hora de inicio de procesamiento
  • Hora de fin de procesamiento
  • Número de identificaciones de entrada
  • Número de registros procesados
  • Número de registros no procesados
  • Errores
    • Id procesado
    • Descripción del error
Criterios de Aceptación
  • El proceso batch usa el validador PESVNO para el archivo de identificaciones de entrada
  • El número de registros generados en el archivo de salida sumado al número de registros inconsistentes generado por PESVNO debe ser igual al número de registros del archivo de entrada.
  • Los valores numéricos decimales usan el caracter decimal coma (,)
  • Los campos del archivo de salida están separados por punto y coma(;)
  • El cálculo de los campos esta basado en el proceso guia icest81.cbl
