rt passing data to handlers

Try using jQuery.on data argument: 

.on( events [, selector ] [, data ], handler )

 The handler seems to have an additional argument named data that points to the data passed.

It should be an object, so it won't be taken by the selector argument.

In all handlers I set $.fn.rt.settings as the data argument, thus: 

  RT.$content.on(
    'keydown',
    $.fn.rt.settings,
    function(e) {
      var key = e.which;

and the handlers automatically have access to everything. In the handler body I replace all references to RT by data, or do 

var RT = data;
