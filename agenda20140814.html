cross reference

Project: build a references tree, where a "reference" is a function call, for the whole code pack.

Surprisingly, the parser might work as a gs program, which would void the need to export the source code.

  1. Run a gs script that exports the source code in a single text file, with a means to know the original source name. Allow for selecting only some of the files.

    For example, intermingle comments marking the start of each component.

    This pass mihjt not be necessary, as esprima runs as a gs program (so far).

    Also, the .gs files are exported (downloaded) in JSON format with all non-ASCII characters expresses as Unicode escapes.

  2. Process the js (actually, gs) sources with the parser (esprima?) and obtain the syntax tree.
  3. Travel the syntax tree and record all the FunctionDeclaration node names, this is the callees list.
  4. Travel the syntax tree and record all the calls, both in the caller (a tree) and in the callees list.

    The callers tree tells, for each function, what are the other functions it calls.

    The callee list tells, for each function, from where is it called.

    Beware of recursion.

    In both cases, it might be of interest to tell if the call is intra- or extra- module.

    In this step the tree must recover the names of the members (libraries, modules), that were somehow recorded in the export step.

  5. Output the call tree and call references in HTML format.
  6. Upload the generated HTML to the drive, with the addition of a quick search filter.
