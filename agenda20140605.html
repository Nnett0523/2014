CARGAJUR

Tratamiento de cada registro de JURAD

  1. Los IDs del archivo de entrada se pasan por el programa PESVNO, y se usa el archivo .inc de inconsistentes para cargar una lista de los IDs con error y el código de error asociado.

    Esto se hace para que la validación sea consistente con los demás procesos, y para que cuando evolucione, la evolución se refleje automáticamente en este proceso.

  2. Por cada registro del archivo de JURAD:
    • Se controlan todos los datos formalmente, si alguno es inválido se descarta el registro totalmente (demandas y embargos).

      Estos controles incluyen que todos los datos numéricos contengan números, que las fechas tengan valores dentro de rango, que el nombre no esté en blanco.

    • Si el ID figura en la lista de IDs con error detectados por el PESVNO ...
      • Si el tipo de error del PESVNO es 03, 04 o 05 es una cuenta sin validar, "tipo 7".

        Se denuncia el error de la misma manera que los demás errores del programa.

        La información se graba en el BDIIALE igual que para los registros correctos. Se emite un error tipo "Cuenta sin validar (tipo 7)".

      • Si el error del PESVNO es otro valor ...

        Se denuncia el error de la misma manera que los demás errores del programa.

        La información se aplica al BDIIALE igual que para los registros correctos.

    • Si el ID no figura en la lista del PESVNO se insertan alertas en BDIIALE normalmente.

      El registro se cuenta como bueno.

      Se impactan una o dos alertas, según el registro contenga demandas y/o embargos.

      • Si el registro de entrada es un movimiento tipo "I" (inserción) y ya existe una alerta con la misma clave, se denuncia un error y la alerta no se inserta. La inserción se intenta para demandas ("J001") y para embargos ("J002") independientemente.
      • Si el registro de entrada es un movimiento tipo "M" (modificación) y no existe una alerta con la misma clave, se denuncia un error y la alerta no se modifica. La modificación se intenta para demandas ("J001") y para embargos ("J002") independientemente.
      • Si el registro de entrada es un movimiento tipo "E" (eliminación) y no existe una alerta con la misma clave, se denuncia un error y la alerta no se elimima. La eliminación se intenta para demandas ("J001") y para embargos ("J002") independientemente.
