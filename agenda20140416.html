Como les prometí, les envío un ejemplo ya implementado de un requerimiento, 
para que puedan observar la forma en que se incluye en los componentes unas
características nuevas. El ejemplo que le envío es del cliente Finesa
(Personalización 22) y el requerimiento es el 16454, en los copys la marca 
de este ajuste esta señalado como R16454.
 
1.       Se recibe el pseudocódigo (Características Finesa CQ16454.docx) y 
mapeo (Sol-Caract_Finesa_Nov142014.xlsx).
2.       El código se incluye en el copy CARACTPER-022.PROC.
3.       La declaración de variables se incluyen en el copy 
         CARACTPER-022.WS.
4.       El mapeo de las características se incluyen en el archivo
         CARACTERISTICAS.DAT.

 
Dado que la personalización de Finesa ya existe, con estos 4 puntos es 
suficiente para que el programa ICGKAR calcule y envíe estas características.
 
Para el proceso de certificación, es necesario “crear” un programa que genere 
un archivo de salida indicando el resultado de cada característica, como les
comente, dicho programa ya existe, simplemente hay que hacer una copia con 
otro nombre y realizarle 3 ajustes:
 
1. Cambiar el FD del archivo de salida (SALIDA-PER22), se modificarían 
   únicamente las variables de nivel 03, ya que las de nivel 02 es la
   identificación del cliente consultado.

2. Mover a la variable NUM-PERSO-CARACT el número de la personalización, que 
   para Finesa es 22, también en todo sitio que pregunte por dicha variable,
   se debe validar que sea igual a 22.

3. En el PERFORM 1000-PROCESAR, mover la variable calculada en el copy
   (01MOR998MX022) a la variable de salida del programa (SLD-01MOR998MX022),
   deben tener la misma longitud.

 
Les envío el programa con el que se realizó la certificación de este tema 
que es el CARACTPER-PER22-16454.CBL, con un ejemplo de un archivo de entrada
(CEDULAS-IMSG) y un archivo de salida (CEDULAS-IMSG-PER22), procesado con 
este programa.
 
Una vez terminado y probado el desarrollo, los entregables son:
 
1.       Programa para certificación.
2.       Orden de proceso (Orden-de-proceso-CQ16454.docx), descripción de la
         ejecución del programa, ya que el programa es enviado a Calidad para 
         que ejecuten el proceso y envíen el archivo de salida a DA para
         certificación.
3.       Layout de salida (Layout-Caracteristicas-CQ16454.xlsx), descripción 
         del archivo de salida.
4.       Lista de componentes modificados con su respectivo MD5.
5.       Diferencias entre versión anterior y versión nueva de cada componente
         modificado.
6.       Soporte de pruebas.
 
Quedo atento a cualquier duda.
