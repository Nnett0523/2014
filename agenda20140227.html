moodle setup

Steps to set the test moodle server in my Windows7 PC "GLB-D1984":

Installation

  1. Download a Windows package, including the whole XAMPP stack (not recommended for production servers), latest was version 2.6.1+ stable.
  2. Open the Complete install packages for Windows page with datailed instructions
  3. I expanded the downloaded tarball in C:\moodle
  4. My C:\moodle directory contains 2 exe files, to start and stop it all. I created links to these in the desktop.
  5. Run a browser and point it to localhost or http://127.0.0.1, it will start asking questions.
  6. Leave the default language "en" for English, and continue: the server runs a series of tests checking that the full stack is available.
  7. In the locations page, don't set the server address to "localhost", use your computer's name, thus: glb-d1984.globant.com (with the right number instead of "1984"). If you leave "localhost" the server can only be accessed fom your PC.
  8. In the database page, set a password, I set GlbStaffing.
  9. The server runs a number of tests andafter it gets back to the questions thing with the administrator data. I set the password GlbStaffing-2014 because it requires characters from all different classes.

    I also entered the Buenos Aires location data.

  10. Next are the "front page" settings. The "front page" is what a user sees upon entering moodle.

    The "full site name" is used in the browser tabs.

    The "short name" appears at the top of all pages.

    The "front page description" is shown as a block at the right of the front page. It was "Knowledge for globers by globers. This is what makes us stand out: we are not afraid of learning""

  11. After this page you are in moodle.

It there was trouble the install page recommends the Amazon version.

Configuration

The configuration is horrible, the purpose of this document is to document it in detail.

Users

I enabled the self-registration feature and "self-registered" two new users without admin permissions: tester and tester2.

The email registration confirmation didn't work (no email configuration was set) so I confirmed them as admin in ADMINISTRATION → Site administration → Users → Accounts → Browse list of users → (small cog icon)

In ADMINISTRATION → Site administration → Plugins → Authentication → Manage authentication the item "Email-based self-registration" has to be enabled, that is, it has to show a non-slashed eye icon. "Email-based" means that the email address is used to identify the user, rather than a user id; this precludes the creation of two or more users with the same email.

Below this, under "Common settings", the "Self registration" option has to be set to "Email-based self-registration".

After all this, the home page shows a registration option.

Ideally all globers should be pre-registered and allowed access by being logged into the Globant domain or by entering the Globant domain password.
