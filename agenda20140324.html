reviving MongoDB

After changing the filesystem layout, moving /home to the ex /D, Mongodb started with: sudo service mongodb start and should stop with sudo service mongodb stop.

The data path id /data and the directory has chmod 777.

The options, from /etc/mongodb.conf: 

options: { 
  config: "/etc/mongodb.conf", 
  dbpath: "/data", 
  logappend: "true", 
  logpath: "/var/log/mongodb/mongodb.log", 
  nojournal: "true", 
  smallfiles: "true" 
}

The data that was in /D is not available and will delete it.
