Public Key Authentication

Git allows you to authenticate via key tokens. This is a powerful authentication mechanism but requires extra configuration at start-up.

Using an User's Private Key to Authenticate

Command Line

You can use the ssh-agent to define a context with a private key in order to execute git commands:

ssh-agent bash -c 'ssh-add <PRIVATE KEY FILE>; <GIT COMMAND>'

The following line uses the private key file 'myuser.pem' from the current directory to clone 'myproject'.

ssh-agent bash -c 'ssh-add myuser.pem; git clone myuser@myserver.com:myproject'

Furthermore, with ssh-add, you can add the private key permanently to your ssh agent.
