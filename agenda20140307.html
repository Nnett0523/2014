Quizz types [MOVED]

  1. One answer out of several (radios)
  2. Zero or more true statements, out of set (checkboxes)
  3. Match enumerated answers with their corresponding questions with DnD or selecting the question and then clicking the answer.
  4. Enter a (rather) exact text (possibly more then one), with some elasticity
    1. with or without quotes
    2. normalize blank space
    3. case-insensitive (optional)
    4. strip accents

    This can be achieved by massaging the text and/or providing a set of valid answers.

    It should be possible to set a "never mind" answer for a particular question that accepts any answer.

    Also, consider symbolic answers, like values from system variables or collected from previous quizzes.

    Allow for checking if the values are numeric and doing some calculations on then in order to check the correctness.

  5. Allow entering some JavaScript or Java code and to run it, somehow sandboxed, and use the output as the result to be checked, maybe the console log, or the content of a specific variable.

 

Progress

This 3-days week was an insights week.

Exams

Firstly, I realized that we need good exams better than courses. For the Globant's purpose it doesn't matter what the globers do but what do they know.

Moreover, we'd like to assess the globers knowledge in an unbiased way, independent of the varying levels of self-esteem.

Tracks

As an initial approach, a learning track on a subject is a tree of exams (minicerts) where the glober starts at the root and has to move to the branches and the leaves.

The globers advance by approving exams. We don't care that much about how they acquired the knowledge; we yes care about having good exams allowing us to assert the globers level.

Actually, a glober that knows it all can blaze the trail from start to end in a few hours just by approving all its exams.

So a track is its exams, and for each one a set of suggestions on how to get the knowledge needed to approve it.

The exams are organized hierarchically: to be able to take a test the globers must have approved al its dependencies.

Course materials

The exams approach doesn't mean that we won't develop any courses. It means that for each leap we'll do some scouting in order to find appropriate trainings, and if there are not then we'll have to make our own course.

As much as we can, we'll cater from the public courses offering from first class universities.

Additional knowledge

For example, the course "M101J: MongoDB for Java Developers" is starting by Mar 17, 2014. By March 27 there will be a course for node.js developers and another for DBAs. These are 5 hours per week courses that span about 8 weeks.

Amazon is offering their "AWS Partner Webinars".

Depending on what we expect to be requested by our clients we could be building knowledge since today, allowing the globers to take the lengthy courses like Mongo's as a part-time additional activity, that can be used to neasure the glober's curiosity and interest.

Exams creation

We should have a means to create good online exams and to allow the globers to take them and know the result immediately.
