-ms-clear

The ::-ms-clear CSS pseudo-element represents a button (the "clear button") at the edge of a text <input> which clears away the <input>'s current value. This button and this pseudo-element are nonstandard and specific to Internet Explorer 10+, hence the vendor prefix. The clear button is only shown on focused, non-empty text <input>s.

We should set:

input[type=text]::-ms-clear { width:0; height:0; display:none; }

It should be set at the top level so it applies to all the input tags in the UI, the effect is to avoid displaying IE's clear button in focused input boxes.

See the -ms-clear options is MSDN.

The problem is that, when emulating IE8 with IE11, the [x] shows but it doesn't show in a real IE8, because this is a "feature" that started as of IE10.

The moral is that we should prevent IE10+ to display this [x] by setting the appropriate styles as a reset item.
