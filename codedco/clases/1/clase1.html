La Internet

La "internet" es una "red" (en inglés "net") de conexiones de datos. Tenemos otras redes que nos proveen otros servicios, como la red eléctrica que conecta una usina donde se genera la electricidad con las casas de la gente, o la red telefónica que conecta a las personas entre sí a través de una central y les permite hablar.

En el caso de la red eléctrica existe la usina, la línea de transmisión que lleva la electricidad de un lado al otro, y la casa donde la electricidad se usa.

La Internet es algo similar. Existen los "web servers" donde viven los datos, la red de transmisión, y la computadora donde usamos los datos.

Visualizamos la red como una nube: no vemos lo que tiene adentro (ni nos interesa normalmente), y cualquier parte donde nos conectemos da igual. Ultimemente ni siquiera es necesario un cable para hacer conexión.

El browser

Es el programa instalado en la PC que usamos para entrar a la Internet.

Hay varios, los más usados son

  1. Google Chrome
  2. Mozilla Firefox
  3. Opera
  4. Microsoft Internet Explorer
  5. Safari

Se supone que todos hacen lo mismo, que muestran las "páginas" exactamente de la misma manera. En la realidad hay pequeñas diferencias que se van reduciendo paulatinamente.

Hay una institucion llamada "World Wide Web Consortium" o "W3C" (3 letras "w" y una "c") que trabaja definiendo "standadrs", definiciones detalladas de cómo debe aparecer cada cosa en la pantalla (en el curso iremos viendo qué cosas).

Las páginas web son "archivos de texto", es decir, contienen solamente letras, números y símbolos (lo que hay en un teclado de PC), y se pueden editar con programas como Notepad o vim. El browser interpreta ese texto según las normas del W3C, lo combina com imágenes (también videos y otros tipos de material) y lo despliega en la pantalla.

El server

El server es una computadora que "está en la Internet". Lo que hace que sea un "web server" y no una computadora cualquiera es que siempre está ejecutando un programa que se llama "web server" (también).

La función del web server es recibir los pedidos de páginas "requests" de los browsers y despachar las páginas que corraspondan.

Los requests de los browsers tienen un formato especial (también definido por el W3C) y se llaman "URLs". Por ejemplo para ver la página de búsqueda en Google el URL es:

https://www.google.com/

¿Cómo funciona?

El browser y el web server se comunican usando la Internet. Para eso usan unos "protocolos" que se llaman HTTP o HTTPS. Un protocolo es una serie de reglas detalladas de qué hacer en todas las situaciones de una relación entre personas, países, o computadoras.

En la imagen se vé al comienzo del URL, el protocolo "https":

El protocolo HTTP es para comunicaciones comunes, y el HTTPS para comunicaciones encriptadas, mensajes en código, casi imposibles de descifrar. Y el protocolo TCP/IP para que las diversas computadoras puedan hablar HTTP entre sí.

Los protocolos de Internet son los que hacen la magia de conectar vía Internet un browser de un lugar con un server que puede estar en el otro lado del mundo.

Por ejemplo http://www.ichikawafm.co.jp es la radio de Ichikawa en Japón. El URL termina con ".jp" en vez de ".ar" como los de Agentina.

Si ponemos el URL de la home page de la radio, la página aparece en el browser! ¿Cómo es posible, cómo funciona, si hay millones y millones de servidores web en la Internet, cómo va justo al que uno quiere?

No es magia. Vamos a ver una explicación simple.

Nosotros, los humanos, entendemos nombres de servidores web como por ejemplo el de Google (google.com) o el de facebook (facebook.com).

Estos nombres de servidores se llaman "domains" (dominios). Los humanos hablamos entre nosotros usando los "domain names", por ejemplo decimos "entré a facebook".

Los programas de computadora usan en vez las direcciones "IP" de los servidores, que son unos números. Por ejemplo la dirección IP del server de la radio de Ichikawa es 211.127.173.50.

Esto se puede comprobar consultando la información sobre la IP 211.127.173.50.

El browser averigua la IP de la radio fácilmente, consultando un servidor "DNS", un diccionario gigante que dice cuál es la IP de cada DNS. Entonces cuando el browser manda el request lo dirige a la dirección IP, no al domain name. Nosotros podemos averiguar la IP usando el comando ping:

Hay otro diccionario, no tan gigante, que dice para dónde ir. El request va de "hop" en "hop" (un "hop" es un salto) saltando de un "router" a otro y aproximándose así a destino. Primero averigua cuál router le conviene para una IP que comienza con 211, el request salta a ese router. Una vez ahí averigua cuál router le conviene para IPs 211.127... y así sucesivamente.

El lenguaje

Cuando un web server recibe un request responde normalmente devolviendo un texto que está escrito en un lenguaje llamado "HTML". Empezamos ahora a aprender cómo usarlo.

El web server que recibe el request responde enviando HTML que está en un archivo, o HTML que arma con información de una base de datos.

Por ejemplo si consultamos twitter para ver lo que hay relacionado con un tag cualquiera, por ejemplo #pizza, el server hace una consulta a una base de datos y arma una página especial para cada request: páginas dinámicas.

Nosotros vamos a hacer páginas estáticas para empezar, páginas que están registradas en un archivo.
