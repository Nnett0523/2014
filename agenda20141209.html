border radius

options

PIE is an HTC script that makes the borders on the fly. It is known to fail sometimes. It injects into tha application an extraneous body that might impact it in unknown ways. It requires the target elements to have relative positioning.

Rounded Corner Image Generator is IMO the best bet. Can be used to generate background images of the border over a transparent background.

jQuery Corner Plugin is the typical definitive plugin, with waaay more options than needed for the modest requirementes of this application. Would inject 250+ js code lines, and require to initialize the elements on page load. Also, it rounds blocks, not borders, so it seems to require two blocks, one inside the other, in order to draw a border.

IE and rounded corners is a blog post suggesting to place small images in the block's corners. It makes rounded rectangles, not borders, requiring element inside element as the jQuery Corner Plugin in order to simulate borders.

Support

We should add a number of prefixed properties if we were to support not-so-new browsers that can do rounded corners but don't understand the standard corner-radius declaration.
