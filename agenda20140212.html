Orion UI comments

  • https://orionhub.org/settings/settings.html#,category=editorSettings
    1. Shows an "Invalid save interval" when the value is high (30s) but the top limit is not stated anywhere

      Suggestion: Give the user freedom to set the desired limit. If for some reason the limit is needed that at least show it in the error message (last resort). Better if shown when the input fields gets focus.

    2. All the inputs (except checkboxes) are shown with the same width.

      Suggestion: This is a usability issue. The size of the input control is a hint for the the length of the expected content.

    3. The [Update] button gets enabled whenever I click, even in another application.
  • https://orionhub.org/settings/settings.html#,category=plugins
    1. The action for installing a plugin is labeled "Submit", which used to be the button type used in forms. This is a generic denomination and thus, less informative.

      Suggestion: Label the link as "Install".

    2. When a plugin is installed, there is no feedback on the outcome of the installation. As the plugin entry is added at the bottom of the list, one has to scroll to the bottom to check what happened.

      Suggestion:

      Organize the plugins in categories, being one the standard 17 basic plugins, and allow for folding the categories lists individually.

      Show only the plugin name together with a disclosure button, so the list is compact and the user doesn't has to scroll back and forth.

      Also, show the optional user.added plugins before the standard ones, most recent first.

  • Wiki
    1. Separate the development stuff (commiters, developers, and the like) from the users stuff.

      This is the most important move in order to have a decent user documentation rather soon; it was a problem with the Eclipse site that might have scared many users-to-be and maybe even make them consider a commercial IDE.

      For example, as a user currently not involved in developing Orion, the "Main Page" link should never drop me in a page with links to "Architecture Council" or "New Commiter Handbook".

    2. Get a link to Orion in http://wiki.eclipse.org/Main_Page ASAP
    3. When using Orion the "Main Page" link should take the user to http://wiki.eclipse.org/Orion, not http://wiki.eclipse.org/

      Prepend "Eclipse" or "Orion" to the links.
