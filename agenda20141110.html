Testing Asynchronous JavaScript by Pete Hodgson

CP issues

  • In the expanded clients list the link to open a client is just the client name.

    It should be the whole cell (<a style="display:block">...)

  • In the expanded clients list highlight the client row on hover.
  • The index.html file does not specify the language.
  • The index.html <head> has relative references like href='../common/stylesheets/popupServiceAlert.css' that break the application if the page is moved.

    The href URIs should start with "/".

  • The "Search by filter ..." prompt is geek lingo.

    It could reflect the choice of the dropdown at the left, like "Search by client name" or "Search by account number", etc.

  • The search artifacts should have tooltips.
  • When searching for clients (by name) clicking the "No results matching search criteria" correctly produces a "There are no results matching search criteria" banner, but the search text is overlaid by another "No results matching search criteria" message.

    The user input should be respected, left unchanged.

  • In the clients search dropdown, the mouse left button click should be ignored, it must not trigger the click action. (event.type === 'mousedown' && event.which === 1 )
  • In the selection of clients by name, the cursor pointer should be the lilnk hand when hovering one of the client names in the list (and not the other elements).
